# Ответы на вопросы преподавателя

## Это настоящий антивирус?

Нет. Это учебный безопасный проект. Он показывает архитектуру GUI + Windows Service + RPC, формат базы, проверку сигнатур, обновления, recovery, мониторинг и инсталлер.

## Где выполняется сканирование?

Сканирование выполняется в Windows-службе. WinUI только отправляет RPC-запросы и показывает результат.

## Почему используется `std::map`?

По ТЗ нужен поиск по первым 8 байтам сигнатуры через красно-чёрное дерево. В стандартных реализациях `std::map` обычно реализован как сбалансированное красно-чёрное дерево. В проекте ключ — первые 8 байт, значение — массив записей с таким префиксом.

## Зачем добавлен Ахо-Корасик?

`std::map` закрывает обязательное требование по префиксу и RB-tree. Ахо-Корасик добавлен как дополнительный быстрый поиск нескольких demo-сигнатур за один проход. Если построить автомат невозможно, остаётся fallback через `std::map`.

## Что хранится в записи базы?

`AvRecord` хранит:

- первые 8 байт сигнатуры;
- длину сигнатуры;
- hash сигнатуры;
- диапазон допустимых смещений;
- тип объекта;
- имя угрозы;
- подпись самой записи.

## Как проверяется база на диске?

Файл `avdb.bin` имеет magic, version, дату выпуска, количество записей, подпись manifest и подписи отдельных записей. Если manifest повреждён, служба принудительно обращается к mock update server. Если повреждена отдельная запись, служба запрашивает замену этой записи с mock update server.

## Почему mock update server — это папка?

Чтобы показать обновление и recovery без внешнего интернета и нестабильных API. Для учебной защиты это проще проверить и воспроизвести.

## Что происходит без лицензии?

`FeatureGate` блокирует сканирование, мониторинг и расписание. GUI получает состояние через RPC и отключает кнопки.

## Как работает расписание?

GUI вызывает `AvStartScanSchedule`. Служба запускает управляемый поток, который по интервалу сканирует выбранный файл, папку или все несъёмные диски. Последний запуск и последний результат доступны через `AvGetScanScheduleStatus`.

## Как работает мониторинг?

GUI выбирает папку, а служба запускает `DirectoryMonitor`. При изменении файла служба сканирует изменённый файл и пишет результат в лог.

## Как останавливается служба?

Перед остановкой GUI показывает Secure Desktop confirmation. После подтверждения отправляется RPC `AvRequestServiceStop`. Служба закрывает свои GUI-процессы и завершает RPC server.

## Зачем DACL hardening?

Это дополнительный учебный пункт: показать ограничение прав на завершение процессов службы и GUI. Это не скрытие процесса и не обход защиты Windows.

## Что показывает инсталлер?

Инсталлер ставит VC++ Runtime, Windows App Runtime 2.0, GUI, службу, документацию и демо-скрипты, регистрирует службу на автозапуск и создаёт uninstall.
